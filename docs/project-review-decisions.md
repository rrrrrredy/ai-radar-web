# 项目回顾决策摘要

本文件只保留历史回顾中与当前发布候选有关、可公开提交的产品决策。原始回顾材料可能包含内部地址或过时要求，不进入仓库，也不作为当前验收依据。

## 继续采用

- AI Industry Radar 是事件级 AI 行业情报产品，不是新闻或 RSS 平铺列表。
- 公开站无需登录；来源、证据、新鲜度、局限和报告质量必须可审计。
- 产品提供中文和英文界面，默认中文；英文原文标题、公司、模型和来源名保持原样。
- 抓取、理解、聚类、评分、报告和发布分层执行；确定性规则负责公开边界与质量门槛。
- DeepSeek 仅用于有界的数据理解与报告生成，默认不因存在密钥而自动在线调用。
- 来源应有分层权重、失败原因、手动阻塞状态和覆盖率统计。
- 不足质量门槛的报告不得包装成完整日报或周报。

## 由当前发布合同覆盖

- 当前发布只保留 `workflow_dispatch` 手动刷新，不启用定时写入。
- `/ask/` 保留为公开证据查询入口；`/write/` 与写作 API 已从当前公开产品退役。
- 登录、收藏、批注、私有来源和后台完整工作流不作为本次公开发布阻塞项。
- X 与微信不自动抓取，来源健康检查不执行写入。
- Cloudflare Pages 是主站；Vercel 仅作为动态参考环境。

## 当前执行原则

- 以本次 final release candidate 验收条件为唯一发布标准。
- 公开快照采用字段白名单，生产导出必须来自公开 Supabase 读取路径并在异常时失败关闭。
- 事件合并优先使用确定性特征，并保留防止跨公司、跨模型和跨事件过度合并的测试。
- 每次发布必须通过数据、安全、构建、部署和对抗审查，并由同一冻结提交生成工件与文档。
